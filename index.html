Sweeten your banking exports into one clean schema.

MapleSchema takes CSVs and raw transaction feeds from different institutions and turns them into a single, predictable JSON model — so you can ship analytics, budgeting, and reporting features without hand-parsing every bank.

One schema, many institutions

Different banks describe the same transaction in wildly different ways. MapleSchema flattens that chaos into a consistent shape so your code doesn’t need a branching tree of special cases.

Built for developers

Explicit direction fields, normalized amounts, clear timestamps, and opinionated naming. No guessing which sign means "debit" this time.

Designed for open banking 2026

Start integrating exports today and be ready when Canadian open banking APIs land. MapleSchema gives you a stable target while the ecosystem matures.

Example: from raw CSV to normalized JSON

Raw input from one institution:

"2024-07-12","-18.42","CAD","POS 07/12  UBER *TRIP   HELP.UBER.COM CA  800-111-1111"

Mapped into a MapleSchema transaction:

{
  "transaction_id": "txn_123",
  "posted_at_utc": "2024-07-12T14:03:11Z",
  "amount": 18.42,
  "direction": "debit",
  "currency": "CAD",
  "merchant_name": "Uber",
  "merchant_category": "Rideshare",
  "raw_description": "POS 07/12  UBER *TRIP   HELP.UBER.COM CA  800-111-1111",
  "source_institution": "Example Bank"
}

Full field list and JSON schema are being published incrementally on GitHub as the project evolves.
